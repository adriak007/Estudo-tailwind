# 13 - Revisao Rapida e Flashcards

## Como revisar

A melhor revisao para Tailwind nao e reler passivamente.
Voce precisa se testar.

Use este arquivo assim:

1. leia a pergunta
2. responda sem olhar outro arquivo
3. so depois confira

## Flashcards

### 1. O que significa utility-first?

Resposta curta:

Construir a interface combinando classes pequenas de responsabilidade unica.

### 2. Quando usar `flex`?

Resposta curta:

Quando a relacao entre elementos e mais linear, em linha ou coluna.

### 3. Quando usar `grid`?

Resposta curta:

Quando ha colunas e linhas mais previsiveis, especialmente para varios cards.

### 4. Qual a ordem mental para ler classes?

Resposta curta:

Layout, tamanho, espacamento, tipografia, visual, estados.

### 5. O que significa mobile-first?

Resposta curta:

Escrever primeiro a base para telas menores e depois ampliar com breakpoints.

### 6. Quando `group` ajuda?

Resposta curta:

Quando um estado no pai precisa afetar filhos.

### 7. Quando `peer` ajuda?

Resposta curta:

Quando um elemento precisa reagir ao estado de um irmao.

### 8. Quando customizar tema?

Resposta curta:

Depois de dominar a base e identificar padroes reais do projeto.

### 9. Quando extrair componente?

Resposta curta:

Quando a repeticao virou padrao real e previsivel.

### 10. Quando usar valor arbitrario?

Resposta curta:

Quando a escala padrao nao cobre um caso especifico e recorrente o bastante para justificar.

## Revisao de 5 minutos

Se tiver pouco tempo, revise so isto:

- `flex` vs `grid`
- `p-*`, `gap-*`, `max-w-*`
- `text-*`, `bg-*`, `shadow-*`
- `md:*`, `lg:*`
- `hover:*`, `focus-visible:*`, `group`, `peer`

## Checklist de nivel

### Nivel iniciante

- consigo montar card e botao
- sei centralizar conteudo
- sei usar `flex` basico

### Nivel intermediario

- monto layout responsivo
- uso `grid` e `group`
- extraio componentes simples

### Nivel avancado

- uso variantes com criterio
- resolvo casos com seletor arbitrario
- mantenho legibilidade e consistencia

## Perguntas finais para voce mesmo

- eu consigo recriar uma tela sem copiar?
- eu dependo demais de cola?
- meu codigo esta legivel?
- meu layout quebra em mobile?
- estou extraindo componente na hora certa?
